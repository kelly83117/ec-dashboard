找不到信？直接點：

⚠ 還沒有 GitHub 帳號？先去 github.com/signup 註冊一個（免費）
2

安裝 Git

⏱ 3 分鐘

Git 是版本控制工具，讓我們的程式碼可以多人同步。

下載 Git for Windows →
安裝步驟：
  • 下載完打開 .exe 檔
  • 所有選項都用預設值，一直按「Next」到底
  • 最後按「Install」→ 完成「Finish」
💡 不用記任何指令，Claude 都會幫你打
3

安裝 Claude 桌面版

⏱ 5 分鐘

用 Claude 跟程式碼對話，跟它說要改什麼它就幫你改。

下載 Claude →
安裝後：
  • 用你的 Email 註冊 / 登入 Claude 帳號
  • Pro 方案 $20 美元/月（建議，使用量比較不會超）— 但免費也能用
4

把專案抓下來

⏱ 3 分鐘

打開 Claude 桌面版 → 新對話 → 完整貼上下面這段話：

我要協作 https://github.com/kelly83117/ec-dashboard 這個專案，請幫我做以下事情： 1. 用 git clone 把專案抓到我的桌面 2. 設定好 git 的使用者名稱跟信箱（我的 GitHub 帳號是：[請改成你的GitHub帳號]） 3. 確認我可以 git push（如果需要登入幫我引導） 4. 之後我要改任何東西、push 上 GitHub，你都直接幫我做
⚠ 記得把方括號 [請改成你的GitHub帳號] 換成你自己的 GitHub 帳號名稱

Claude 會跟你聊一下、確認一些設定、然後完成整套流程。第一次 push 時可能會跳出登入視窗，按它指示登入 GitHub 就好。

5

第一次改東西（測試）

⏱ 2 分鐘

試一下：跟 Claude 說

先 git pull 拉最新版 然後幫我把首頁 title 改成「測試 - 你的名字」並 push 上 GitHub

等 1-2 分鐘 → 打開 儀表板網址 強制重整（Ctrl+F5）→ 看到分頁標題改了就成功！

記得測試完跟 Claude 說「改回原本的 title」並 push，把測試還原。
✓

日常工作流程

之後每次要改儀表板，照這個流程：

  1. 跟 Claude 說：「先 git pull 拉最新版」
  2. 講你要改的東西（例：「幫我把營收輸入框改大一點」）
  3. Claude 改完會幫你 commit + push
  4. 等 1-2 分鐘 GitHub Pages 部署，刷新網址 就能看到
🚨 最重要：不要跟 Kelly 同時改！
在 LINE 群組說一聲「我要改囉」「我改完了」，輪流動工。同時改可能會 merge conflict（衝突），雖然 Claude 也能處理，但能避免就避免。

❓ 常見問題

Q：我怕弄壞東西，會不會把現有的儀表板搞爛？

不會。每次改都會生成新的 commit，GitHub 保留所有歷史。出事跟 Claude 說「還原到上一個 commit」就回到原狀。

Q：Claude 跟我說「需要登入 GitHub」怎麼辦？

第一次 push 會跳出瀏覽器要你登入 GitHub。照畫面指示登入就好（會自動記住）。Claude 會引導你。

Q：改完按 push 後，多久能在儀表板看到？

大約 30 秒 ~ 2 分鐘。GitHub Pages 會自動部署。然後在儀表板網址按 Ctrl+F5 強制清快取就能看到新版。

Q：我跟 Kelly 同時都改了會怎樣？

Git 會跳 merge conflict（衝突）。跟 Claude 說「處理一下 conflict」，他會幫你選保留哪一邊。但能避免就避免，請在 LINE 約定。

Q：我電腦上沒有 Claude 怎麼用？

可以用網頁版 claude.ai，但檔案操作需要桌面版才能直接讀寫專案資料夾。建議裝桌面版。

Q：完全卡住怎麼辦？

把 Claude 視窗的對話截圖傳給 Kelly，或直接複製錯誤訊息問 Claude「這是什麼錯誤、怎麼解決」。
